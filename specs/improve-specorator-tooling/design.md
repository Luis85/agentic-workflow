# Design — Specorator improvement command design

Add a new `.claude/commands/specorator/` namespace with four commands. Each command delegates procedural detail to one shared `.claude/skills/specorator-improvement/SKILL.md` file to avoid duplicated guardrails.

## Boundaries

- Commands define entry points, inputs, and "don't" constraints.
- The skill defines the reusable improvement loop and mode-specific checklists.
- Core docs explain when to use the commands.
- Generated command inventories remain generated by `npm run fix:commands`.

## Decisions

- No ADR required; this adds a contributor workflow surface but does not change the constitution, lifecycle stages, or architecture.
